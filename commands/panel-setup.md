# Panel Setup — Render `.env.local` from plugin settings

Reads `.claude/dj-music.local.md` (YAML frontmatter) and writes the values
required by the Next.js panel into `panel/.env.local`. Idempotent — safe
to re-run after editing the settings file.

## Steps

1. Verify the settings file exists. If not, copy the template and stop:

```bash
set -euo pipefail
SETTINGS=".claude/dj-music.local.md"
if [[ ! -f "$SETTINGS" ]]; then
  if [[ -f "${SETTINGS}.example" ]]; then
    cp "${SETTINGS}.example" "$SETTINGS"
    echo "Created $SETTINGS from template. Edit it and re-run /panel-setup."
    exit 0
  fi
  echo "Missing $SETTINGS and no template found." >&2
  exit 1
fi
```

2. Parse the frontmatter and extract the three panel fields:

```bash
FRONTMATTER=$(sed -n '/^---$/,/^---$/{ /^---$/d; p; }' "$SETTINGS")

read_field() {
  echo "$FRONTMATTER" \
    | grep -E "^$1: *" \
    | head -1 \
    | sed "s/^$1: *//" \
    | sed 's/^"\(.*\)"$/\1/'
}

SUPABASE_URL=$(read_field supabase_url)
SUPABASE_KEY=$(read_field supabase_anon_key)
MCP_URL=$(read_field mcp_http_url)

if [[ -z "$SUPABASE_URL" || -z "$SUPABASE_KEY" || -z "$MCP_URL" ]]; then
  echo "Settings file is missing one of: supabase_url, supabase_anon_key, mcp_http_url" >&2
  exit 2
fi
```

3. Write `panel/.env.local`. Never echo the key to stdout:

```bash
mkdir -p panel
cat > panel/.env.local <<EOF
# Generated by /panel-setup from .claude/dj-music.local.md
# Do not edit by hand — edit the settings file and re-run /panel-setup.
NEXT_PUBLIC_SUPABASE_URL=$SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=$SUPABASE_KEY
MCP_HTTP_URL=$MCP_URL
EOF
chmod 600 panel/.env.local
echo "Wrote panel/.env.local (Supabase URL: $SUPABASE_URL, MCP: $MCP_URL)"
```

4. Remind the user that the panel dev server must be restarted to pick up
   the new environment. If it's already running, Next.js hot-reload does
   NOT re-read `.env.local` — a full restart is required.
